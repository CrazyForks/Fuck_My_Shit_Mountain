🔍 详细发现

1. TOTP 时钟偏差窗口已文档化但未实现

Medium Confirmed
📁 Security 📍 auth.rs:366-383 🎯 verify_totp_step / verify_totp_step_at
问题： README 明确声明"TOTP 校验允许前后各一个 30 秒窗口的时钟偏差"，但代码中的 verify_totp_step_at 只检查当前 30 秒步长。如果用户手机时钟偏差超过 ±15 秒，即使输入正确的 TOTP 验证码也会被拒绝，导致管理员被锁在面板外。
// 当前代码：只检查 now_step fn verify_totp_step_at(totp_secret, code, now_step) -> Option<u64> { let expected = totp_code_for_step(secret, now_step); // 没有检查 now_step - 1 或 now_step + 1 } // README.md:534 声称： // "TOTP 校验允许前后各一个 30 秒窗口的时钟偏差"
最小修复
fn verify_totp_step_at(secret: Option<&[u8]>, code: &str, now_step: u64) -> Option<u64> { let secret = secret?; if code.len() != 6 || !code.bytes().all(|b| b.is_ascii_digit()) { return None; } for step in [now_step.saturating_sub(1), now_step, now_step.saturating_add(1)] { let expected = totp_code_for_step(secret, step); if constant_time_compare_bytes(expected.as_bytes(), code.as_bytes()) { return Some(step); } } None }
⏱ 预估工作量： 10 分钟  ·  回归测试建议： 更新 verify_totp_step_accepts_only_current_step 测试，使其接受相邻步长的验证码

2. 生产代码 .unwrap() 在 web_assets.rs

Medium Confirmed
📁 Stability 📍 web_assets.rs:72, 110 🎯 verify_2fa_page / serve_file
问题： 两处 Response::builder().body(...).unwrap() 在生产路径中。虽然实践中很少失败，但一旦触发 panic 将杀死整个 axum 服务进程，导致所有监控中断。
// 第 72 行 builder.body(Body::from(file.contents())).unwrap() // 第 110 行 builder.body(Body::from(file.contents())).unwrap()
最小修复
builder.body(Body::from(file.contents())).unwrap_or_else(|_| { (StatusCode::INTERNAL_SERVER_ERROR, "failed to build response").into_response() })
⏱ 预估工作量： 5 分钟  ·  回归测试建议： 现有的 test_spa_index_exists 测试已可捕获回归

4. SMTP AUTH PLAIN 密码通过 format! 构建

Low Confirmed
📁 Security 📍 smtp.rs:109
问题： 密码通过 format! 拼接到 SASL PLAIN 字符串中。虽然是 SMTP 协议规范行为，但密码以明文 String 存在于内存中，未显式清零。
let payload = STANDARD.encode(format!("\0{}\0{password}", config.username));
最小修复（防御性）
let mut buf = Vec::with_capacity(1 + config.username.len() + 1 + password.len()); buf.push(0); buf.extend_from_slice(config.username.as_bytes()); buf.push(0); buf.extend_from_slice(password.as_bytes()); let payload = STANDARD.encode(&buf); buf.zeroize();
⏱ 预估工作量： 10 分钟  ·  当前风险可接受，高安全场景可引入 secrecy crate
